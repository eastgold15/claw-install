# 🚀 OpenClaw 启动与运行

> OpenClaw 服务的启动、管理和监控指南

## 📋 基础概念

### Gateway 网关
Gateway 是 OpenClaw 的核心后台服务，负责处理所有 AI 请求和任务调度。

### Dashboard 控制台
Dashboard 是 OpenClaw 的 Web 管理界面，提供可视化的系统管理和监控。

## 🚀 启动服务

### 1. 启动 Gateway 网关

```bash
# 启动后台服务（推荐）
openclaw gateway

# 或以后台模式启动
openclaw gateway --daemon

# 指定配置文件启动
openclaw gateway --config /path/to/config.json
```

### 2. 启动 Dashboard 控制台

```bash
# 启动 Web 界面
openclaw dashboard

# 指定端口启动
openclaw dashboard --port 3000

# 后台模式启动
openclaw dashboard --daemon
```

### 3. 快速启动脚本

创建一个快速启动脚本 `start-openclaw.sh`:

```bash
#!/bin/bash
# 启动 OpenClaw Gateway
echo "启动 OpenClaw Gateway..."
openclaw gateway --daemon

# 等待服务启动
sleep 5

# 启动 Dashboard
echo "启动 Dashboard..."
openclaw dashboard --port 3000

echo "OpenClaw 启动完成！"
echo "Dashboard: http://localhost:3000"
```

## 📊 服务管理

### 查看服务状态

```bash
# 查看 Gateway 状态
openclaw gateway status

# 查看所有服务状态
openclaw status --all

# 查看 Dashboard 状态
openclaw dashboard status
```

### 重启服务

```bash
# 重启 Gateway
openclaw gateway restart

# 重启 Dashboard
openclaw dashboard restart

# 重启所有服务
openclaw restart --all
```

### 停止服务

```bash
# 停止 Gateway
openclaw gateway stop

# 停止 Dashboard
openclaw dashboard stop

# 停止所有服务
openclaw stop --all
```

## 🔧 配置管理

### 查看当前配置

```bash
# 查看当前配置
openclaw config

# 查看特定配置项
openclaw config get gateway.port

# 验证配置文件
openclaw config validate
```

### 修改配置

```bash
# 进入交互式配置
openclaw config

# 或直接编辑配置文件
openclaw config edit
```

## 📝 日志管理

### 查看日志

```bash
# 查看 Gateway 日志
openclaw gateway logs

# 查看 Dashboard 日志
openclaw dashboard logs

# 实时查看日志
openclaw gateway logs --follow

# 查看最近 100 行日志
openclaw gateway logs --tail 100
```

### 日志配置

```bash
# 设置日志级别
openclaw config set logging.level debug

# 设置日志文件路径
openclaw config set logging.path /var/log/openclaw.log
```

## 🚨 故障排除

### 1. 端口占用

```bash
# 检查端口占用
netstat -an | findstr :18789

# 查看进程占用端口
tasklist | findstr node
```

**解决方案**: 
- 修改配置文件中的端口设置
- 或停止占用端口的进程

### 2. 服务启动失败

```bash
# 检查服务状态
openclaw gateway status

# 查看详细错误日志
openclaw gateway logs --level error
```

**解决方案**:
- 检查配置文件语法
- 确认依赖服务正常运行
- 查看系统日志获取详细信息

### 3. 网络连接问题

```bash
# 测试网络连接
ping api.z.ai

# 检查防火墙设置
netsh advfirewall show allprofiles
```

**解决方案**:
- 配置防火墙规则
- 检查代理设置
- 确认网络连接正常
