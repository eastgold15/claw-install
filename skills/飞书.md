# 📱 飞书集成配置

> OpenClaw 与飞书的集成配置，实现 AI 助手在飞书中的消息处理

## 📋 集成概述

飞书集成让 OpenClaw 能够通过飞书机器人接收和发送消息，实现 AI 助手在飞书工作区中的智能对话。


## ⚙️ 配置步骤

### 1. 飞书开放平台配置

1. **访问开放平台**: https://open.feishu.cn/app
2. **创建应用**: 选择 "机器人" 类型
3. **获取凭证**: App ID 和 App Secret
4. **配置权限**: 添加上述权限范围
5. **发布应用**: 提交审核并发布

### 2. OpenClaw 配置

```bash
# 配置飞书账号
openclaw config set channels.feishu.accounts.default.appId "your_app_id"
openclaw config set channels.feishu.accounts.default.appSecret "your_app_secret"

# 启用飞书渠道
openclaw config set channels.feishu.enabled true
```

### 3. 绑定智能体

```bash
# 绑定到主智能体
openclaw bind feishu default main
```

## 🔧 高级配置

### 1. 消息策略

```json
{
  "channels": {
    "feishu": {
      "accounts": {
        "default": {
          "dmPolicy": "open",      // 私聊策略
          "groupPolicy": "open",   // 群聊策略
          "allowFrom": ["*"],      // 允许的用户
          "groupAllowFrom": ["*"]  // 允许的群组
        }
      }
    }
  }
}
```

### 2. 连接模式

```json
{
  "channels": {
    "feishu": {
      "connectionMode": "websocket",  // WebSocket 连接
      "domain": "feishu"
    }
  }
}
```

### 3. 显示名称

```bash
# 设置飞书账号显示名称
openclaw config set channels.feishu.accounts.default.displayName "OpenClaw"
```

## 🚨 常见问题

### 1. 权限不足

**问题**: 机器人无法发送消息或访问资源

**解决方案**:
1. 检查飞书开放平台权限配置
2. 确保应用已发布
3. 重新授权应用

### 2. 连接失败

**问题**: WebSocket 连接失败

**解决方案**:
1. 检查网络连接
2. 验证 App ID 和 Secret
3. 查看 OpenClaw 日志

### 3. 消息处理异常

**问题**: 消息发送或接收异常

**解决方案**:
1. 检查智能体绑定状态
2. 验证消息策略配置
3. 查看详细错误日志
